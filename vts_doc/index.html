@verificatum/verificatum-vts

Verificatum TypeScript library (VTS)

Self-contained cryptographic library for use in electronic voting clients.

Usage | Prerequisites | Building | License | Acknowledgments | Contributing

Usage

Compiled versions. Single-file TypeScript and JavaScript compiled versions are provided in ./compiled/ for convenience.

Documentation. Browse the documentation or build it locally with make doc. The documentation links directly to the relevant lines in the actual source code at GitHub.

Examples, tests, and benchmarks. Try the software now, or build the library and start a webserver at http://localhost:8000/index.html using make webserver. The source code of the examples is documented to allow simple cut and paste into your own code.

Comparison with other projects. There are detailed benchmarks and comparisons with other projects in ./benchmarking/compiled/ which can be re-run on any platform. They are described and interpreted in ./benchmarking/README.md.

Prerequisites

Although the software has zero external dependencies we rely on a minimal set of building tools. Installation instructions are given for Ubuntu 22.04. Adapt as needed.

# Building tools.
sudo apt install make m4

# Most recent version of node.
curl -fsSL https://deb.nodesource.com/setup_current.x | sudo -E bash -
sudo apt install -y nodejs

# TypeScript globally.
sudo npm install -g typescript

# Install NPM development dependencies.
npm i

# To enable WASM code the following is needed.
sudo apt install clang lld
sudo update-alternatives --install /usr/bin/wasm-ld wasm-ld /usr/bin/wasm-ld-14 100

# To run the webserver that serves the test and benchmark suites you
# may need to install Python 3.
sudo apt install python3

Building

The software can be configured and built in a number of ways, but the simplest way is make tsmodules or make jsmodules to generate and/or build TypeScript/JavaScript modules, or make tsfiles or make jsfiles to generate and/or build single file versions.

The corresponding test suites can similarly be run with make check_(modules|files), and benchmarks with make bench_(modulesmodules|files).

Read more about the source code and configuration in ./README_DETAILS.md.

The generated TypeDoc pages point to the actual source to help you do this. If you have a better solution to any of the hacks used, then please let us know how our work can be improved.
